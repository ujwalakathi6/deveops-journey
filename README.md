📌 DevOps Learning – Day 1
🔹 Cloud Computing Basics

Cloud computing provides computing services over the internet.

✔️ Deployment Models:

Public Cloud – Available to everyone

Private Cloud – Used by a single organization

Hybrid Cloud – Combination of public and private

✔️ Service Models:

IaaS (Infrastructure as a Service) – Provides virtual machines (Example: EC2)

PaaS (Platform as a Service) – Provides platform to build apps

SaaS (Software as a Service) – Ready-to-use software

🔹 Creating an AWS Account

Steps to create an AWS account:

Go to AWS official website

Click on Create an AWS Account

Enter email, password, and account name

Add billing information (card required)

Verify phone number (OTP)

Choose Basic/Free Tier plan

Login to AWS Management Console

🔹 What is DevOps?

DevOps is a combination of Development (Dev) and Operations (Ops).
It is a set of practices that helps teams build, test, and release software faster and more reliably.

👉 Goal:

Faster delivery 🚀

Better collaboration 🤝

Continuous improvement 🔁

🔹 DevOps Lifecycle

The DevOps lifecycle consists of the following stages:

Planning – Define project requirements

Development – Write code

Build – Compile and package the application

Testing – Test for bugs and errors

Release – Prepare the application for deployment

Deployment – Deploy to servers/cloud

Operations – Monitor and manage the system

Monitoring – Track performance and issues

👉 This cycle is continuous and repeats.

🔹 Introduction to EC2

EC2 (Elastic Compute Cloud) is a service that provides virtual servers in the cloud.

✔️ Key Features:

Launch virtual machines

Choose OS (Linux/Windows)

Scalable resources

Pay only for what you use

✔️ Basic Steps to Launch EC2:

Go to EC2 Dashboard

Click Launch Instance

Choose an Amazon Machine Image (AMI)

Select instance type (e.g., t2.micro – free tier)

Configure settings

Add storage
